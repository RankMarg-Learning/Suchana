# UI Building Golden Rule Book  
Version: 1.0  
Purpose: Production-Grade UI Design Standards for Mobile and Web Applications  
Audience: Product Designers, Frontend Engineers, Founders, UI Architects  

---

# 1. Core Philosophy

## Rule 1.1: UI Exists to Reduce Cognitive Load

The primary purpose of UI is not beauty.  
The primary purpose is clarity and execution.

A good UI answers these questions instantly:

- What is this?
- What can I do here?
- What should I do next?

If the user has to think, the UI has failed.

---

## Rule 1.2: Show, Don't Explain

Avoid instructions whenever possible.

Bad:
"Click here to view exams"

Good:
"View Exams →"

The UI should communicate through structure and visual hierarchy.

---

## Rule 1.3: Every Screen Must Have One Primary Action

Each screen must clearly answer:

What is the main action here?

Examples:

Home screen → View Exams  
Exam screen → View Timeline  
Result screen → Check Result  

Never present multiple competing primary actions.
